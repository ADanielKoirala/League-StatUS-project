# Product Requirements Summary

## Stakeholders

- League organizers / administrators
- Referees
- Players
- Northeast Ohio recreational volleyball community

## Functional Requirements

### Players

- Create and manage basic profile information
- View schedules
- Request team membership
- View team performance information

### Referees

- View assigned games
- Enter and update scores
- View rosters
- Pick up available shifts

### Administrators

- Create and update schedules
- Assign referees
- Manage teams and users
- Review league and referee data

## Quality Requirements

### Usability

The platform should be easy to use for participants with varying technical skill levels and should remain usable on mobile browsers during games.

### Performance

Expected data volume is modest, but the system should support thousands of accounts and routine daily use.

### Reliability

The system should be backed up regularly and restored quickly enough to avoid disrupting scheduled games.

### Maintainability

The system should be modular so future features such as tournament management and substitute-player workflows can be added without rewriting the whole platform.

## Future Work

- Tournament management
- Player substitute hub
- Advanced analytics
- Mobile app version
- Notification delivery for schedule changes
